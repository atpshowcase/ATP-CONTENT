BAB A. Introduction

A.1 Asal-usul C#

C# lahir sekitar tahun 1999-2000 di dalam Microsoft, sebagai bagian dari proyek besar bernama .NET. Waktu itu Microsoft lagi ngebangun platform baru yang bisa nyaingin Java dari Sun Microsystems. Mereka butuh bahasa yang modern, aman dari bug tipe data, dan enak dipakai di ekosistem Windows.

Nama "C#" sendiri diambil dari notasi musik — tanda "sharp" (#) yang artinya setengah nada lebih tinggi. Maknanya simbolis: C# itu satu level di atas C dan C++, lebih modern dari keduanya. Versi pertama resmi dirilis tahun 2002 bareng .NET Framework 1.0.

// C# pertama kali terasa seperti ini (versi 1.0, 2002)
using System;

class HelloWorld
{
    static void Main()
    {
        Console.WriteLine("Hello, World!");
    }
}
Analogi: C# itu kayak brand sepatu baru yang muncul karena ada gap di pasar — ada pemain lama yang udah terlalu ribet (C++) dan ada pemain yang terlalu dikunci ekosistemnya (Java). C# hadir buat ngisi tengah-tengah itu.

A.2 Anders Hejlsberg — Orang di Balik C#

Arsitek utama C# adalah Anders Hejlsberg, programmer asal Denmark yang bergabung ke Microsoft tahun 1996. Sebelum di Microsoft, dia sudah punya track record luar biasa: dia yang bikin Turbo Pascal di Borland, dan jadi lead designer buat bahasa Delphi.

Ketika Microsoft minta dia bangun bahasa baru untuk .NET, Anders ngerancang C# dari pengalaman panjangnya. Dia tau persis apa yang bikin programmer frustrasi di C++ (management memori yang ribet) dan apa yang bikin programmer terkunci di Java (ekosistem satu vendor). C# dirancang buat selesaikan kedua masalah itu.

  • Garbage collector otomatis — programmer gak perlu manual-manage memori.
  • Type-safe — error tipe data ketangkap lebih awal, saat compile bukan saat runtime.
  • Multi-paradigma — bisa OOP, functional, bahkan low-level kalau perlu.
  • Tooling kuat — Visual Studio dan kemudian VS Code jadi backbone development-nya.
Analogi: Anders itu kayak chef yang udah pernah masak di semua jenis dapur. Dia tau resep mana yang gagal dan kenapa, jadi waktu bikin resep sendiri, dia tahu betul mana yang harus dihindari.

A.3 Evolusi C# — Dari Versi 1 Sampai Sekarang

C# berkembang pesat dari versi ke versi, tiap rilis nambah fitur yang bikin kode makin singkat, aman, dan ekspresif. Ini bukan bahasa yang stagnan — tiap 1-2 tahun ada versi baru yang ngerespon kebutuhan developer modern.

  • C# 1.0 (2002) — Fondasi: class, interface, delegate, event.
  • C# 2.0 (2005) — Generics, nullable types, iterator.
  • C# 3.0 (2007) — LINQ masuk, lambda expression, var keyword.
  • C# 4.0 (2010) — Dynamic typing, named/optional parameter.
  • C# 5.0 (2012) — async/await, revolusi pemrograman asinkron.
  • C# 6.0 (2015) — String interpolation ($"..."), null-conditional operator (?.).
  • C# 7.0-7.3 (2017-2018) — Tuple, pattern matching, local function.
  • C# 8.0 (2019) — Nullable reference types, switch expression.
  • C# 9.0 (2020) — Record types, init-only setter, top-level statement.
  • C# 10.0 (2021) — Global using, file-scoped namespace.
  • C# 11.0-13.0 (2022-2024) — Raw string literal, required member, collection expression.
// C# modern (versi 10+) terasa jauh lebih ringkas
// Top-level statement — gak perlu class Program + Main lagi
using System;

Console.WriteLine("C# modern, tanpa boilerplate!");
Analogi: C# itu kayak smartphone yang tiap tahun update OS-nya. Core-nya sama, tapi tiap versi makin smooth, makin fitur, makin enak dipakai. Yang belajar dari awal tetap relevan, karena fondasi lama tetap jalan di versi baru.

BAB B. Computers and Programs

B.1 Computers

Komputer itu mesin yang nurut instruksi. Dia gak nebak, gak improvisasi. Kalau instruksi lo ambigu, hasilnya juga ngaco.

Analogi: komputer itu kayak kasir otomatis. Lo pencet menu salah, yang keluar ya pesanan salah juga.

B.2 Programs and Programming

Program adalah sekumpulan instruksi. Programming adalah proses nyusun instruksi itu biar masalah selesai dengan benar, konsisten, dan bisa diulang.

int harga = 20000;
int qty = 3;
int total = harga * qty;
Console.WriteLine($"Total bayar: {total}");
Analogi: programming itu kayak bikin SOP kerja tim biar shift pagi dan malam hasilnya tetap sama bagus.

B.3 Programming Languages

Bahasa pemrograman itu alat komunikasi antara manusia dan mesin. C# termasuk bahasa high-level yang readable buat manusia, tapi tetap kuat performanya.

  • Readable dan terstruktur.
  • Type-safe, jadi lebih aman dari banyak bug.
  • Cocok buat desktop, web, API, game, sampai cloud.
Analogi: bahasa pemrograman itu kayak bahasa obrolan tim. Kalau satu standar, koordinasi jadi cepat.

B.4 C#

C# itu bahasa modern dari ekosistem .NET. Kelebihannya ada di sintaks yang jelas, tooling matang, dan dukungan enterprise yang kuat.

string nama = "Dita";
int umur = 21;
bool aktif = true;

Console.WriteLine($"User: {nama}, Umur: {umur}, Aktif: {aktif}");
Analogi: C# itu kayak motor harian yang nyaman dipakai buat kerja harian, tapi tetap kuat buat perjalanan jauh.

BAB C. Simple Data Processing

C.1 A First C# Program

Program pertama biasanya sederhana: tampilkan teks ke layar. Tujuannya bukan pamer fitur, tapi ngebuktiin environment lo sudah jalan.

using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Hello, World!");
    }
}
Analogi: ini kayak cek mic "tes satu dua" sebelum acara dimulai.

C.2 Manipulating Data

Data processing dasar mencakup input, proses, output. Lo baca data, olah dengan logika, lalu tampilkan hasilnya.

int nilaiA = 80;
int nilaiB = 90;
double rataRata = (nilaiA + nilaiB) / 2.0;

Console.WriteLine($"Rata-rata: {rataRata}");
Analogi: lo ngitung kas masuk-keluar harian buat tahu untung bersih.

C.3 Writing a Program

Menulis program yang baik itu bukan cuma bikin jalan, tapi bikin jelas alurnya. Biasakan pecah langkah: definisi masalah, desain alur, coding, testing.

  • Tulis tujuan program.
  • Susun langkah logika.
  • Implementasi bertahap.
  • Uji dengan data normal dan data ekstrem.
Analogi: jangan langsung masak rame-rame tanpa resep, nanti dapur chaos.

BAB D. Creating Programs

D.1 Methods

Method itu cara memecah pekerjaan jadi unit kecil yang reusable. Dengan method, kode lebih rapi dan gampang dites.

static int Tambah(int a, int b)
{
    return a + b;
}

Console.WriteLine(Tambah(5, 7));
Analogi: method itu kayak jobdesc tim, tiap orang pegang satu tugas jelas.

D.2 Variables and Scope

Scope ngatur di mana variabel bisa diakses. Pahami scope biar gak ketabrak nilai dari blok lain.

int total = 100;

if (total > 50)
{
    string status = "Lolos";
    Console.WriteLine(status);
}
// Console.WriteLine(status); // error: di luar scope
Analogi: scope itu kayak akses ruangan VIP, gak semua orang bisa masuk.

D.3 Arrays

Array dipakai buat nyimpen kumpulan data dengan tipe sama. Cocok saat lo punya list yang ukurannya udah kebayang.

int[] nilai = { 70, 85, 90, 60 };

foreach (int n in nilai)
{
    Console.WriteLine(n);
}
Analogi: array itu kayak daftar antrean nomor pelanggan.

D.4 Exceptions and Errors

Error pasti ada. Poin pentingnya adalah handling yang aman supaya program gak langsung crash tanpa pesan jelas.

try
{
    int x = int.Parse("abc");
    Console.WriteLine(x);
}
catch (FormatException ex)
{
    Console.WriteLine($"Input gak valid: {ex.Message}");
}
Analogi: kalau mesin kopi error, lo butuh SOP penanganan, bukan panik massal.

D.5 The Switch Construction

Switch cocok buat banyak pilihan berdasarkan satu nilai. Lebih rapi daripada if-else panjang.

string role = "admin";

switch (role)
{
    case "admin":
        Console.WriteLine("Akses penuh");
        break;
    case "staff":
        Console.WriteLine("Akses terbatas");
        break;
    default:
        Console.WriteLine("Akses tamu");
        break;
}
Analogi: switch itu kayak jalur layanan berdasarkan tipe member.

D.6 Using Files

Operasi file dipakai buat simpan dan baca data sederhana dari disk: laporan, log, config, dsb.

using System.IO;

Analogi: file itu kayak buku catatan kas harian yang bisa dibuka ulang kapan saja.

BAB E. Creating Solutions

E.1 Our Case Study: Friendly Bank

Case study dipakai biar lo lihat gimana konsep teknis dipakai buat kasus nyata. Friendly Bank ngajarin design sistem, data model, dan alur transaksi.

// Model dasar Friendly Bank
class AkunBank
{
    public string NomorAkun { get; set; }
    public string NamaPemilik { get; set; }
    public double Saldo { get; set; }

    public void Deposit(double jumlah)
    {
        Saldo += jumlah;
        Console.WriteLine($"Deposit {jumlah:C} berhasil. Saldo sekarang: {Saldo:C}");
    }
}

AkunBank akun = new AkunBank
{
    NomorAkun = "FBK-001",
    NamaPemilik = "Reza",
    Saldo = 500000
};

akun.Deposit(150000);
Analogi: ini kayak simulasi buka bisnis beneran, bukan cuma latihan teori.

E.2 Enumerated Types

Enum bikin kode lebih aman dan mudah dibaca saat nilai validnya terbatas.

enum StatusAkun { Aktif, Nonaktif, Diblokir }

StatusAkun status = StatusAkun.Aktif;
Console.WriteLine(status);
Analogi: enum itu daftar pilihan menu tetap, gak bisa nambah opsi ngaco.

E.3 Structures

Structure cocok buat data kecil yang ringan dan biasanya bersifat value type.

struct Titik
{
    public int X;
    public int Y;
}

Titik p = new Titik { X = 10, Y = 20 };
Console.WriteLine($"{p.X}, {p.Y}");
Analogi: struct itu kayak kartu data ringkas yang gak butuh perilaku kompleks.

E.4 Objects, Structures and References

Pahami bedanya value type dan reference type biar lo gak salah copy data atau mutasi object.

// Value type (struct) — tiap variabel punya salinan sendiri
struct Poin { public int X, Y; }

Poin a = new Poin { X = 5, Y = 10 };
Poin b = a;       // b adalah salinan, bukan alias
b.X = 99;
Console.WriteLine(a.X); // output: 5 — a tidak ikut berubah

// Reference type (class) — variabel berbagi satu objek yang sama
class Kotak { public int Lebar; }

Kotak k1 = new Kotak { Lebar = 50 };
Kotak k2 = k1;    // k2 menunjuk ke objek yang sama dengan k1
k2.Lebar = 200;
Console.WriteLine(k1.Lebar); // output: 200 — k1 ikut berubah
Analogi: value type kayak fotokopi nota (punya salinan sendiri), reference type kayak share link dokumen yang sama.

E.5 Designing With Objects

Desain object fokus pada tanggung jawab yang jelas. Class yang baik itu cohesive dan minim ketergantungan gak perlu.

  • Tiap class punya tujuan spesifik.
  • Kurangi class "serba bisa".
  • Utamakan keterbacaan dan testability.
// Desain cohesive: tiap class satu tanggung jawab
class ValidasiTransaksi
{
    public bool CekSaldo(double saldo, double penarikan)
        => penarikan > 0 && penarikan <= saldo;
}

class TransaksiService
{
    private ValidasiTransaksi _validasi = new ValidasiTransaksi();

    public void Tarik(double saldo, double jumlah)
    {
        if (_validasi.CekSaldo(saldo, jumlah))
            Console.WriteLine($"Penarikan {jumlah:C} berhasil.");
        else
            Console.WriteLine("Gagal: saldo tidak cukup atau jumlah tidak valid.");
    }
}

new TransaksiService().Tarik(300000, 100000);
Analogi: jangan satu orang jadi kasir, barista, security, dan akuntan sekaligus.

E.6 Static Items

Static dipakai untuk data atau method yang milik class, bukan milik objek tertentu.

class Kalkulator

Analogi: static itu kayak aturan umum komunitas, berlaku ke semua member.

E.7 The Construction of Objects

Constructor dipakai saat object dibuat. Cocok untuk inisialisasi data awal yang wajib ada.

E.8 From Object to Component

Saat aplikasi membesar, object-object digabung jadi komponen yang bisa dipakai ulang antar modul.

// Komponen: gabungan object dengan tanggung jawab jelas
class Logger
{
    public void Log(string pesan)
        => Console.WriteLine($"[LOG] {DateTime.Now:HH:mm:ss} - {pesan}");
}

class NotifikasiService
{
    private Logger _log = new Logger();

    public void Kirim(string penerima, string isi)
    {
        _log.Log($"Kirim ke {penerima}: {isi}");
        Console.WriteLine($"Notifikasi terkirim ke {penerima}");
    }
}

new NotifikasiService().Kirim("Adi", "Transaksi kamu berhasil");
Analogi: dari satu barista jadi satu tim operasional lengkap yang punya SOP sendiri.

E.9 Inheritance

Inheritance memungkinkan class turunan mewarisi properti/method dari class induk.

{
    public string Nama = "User";
}

class Admin : User
{
    public void Info() => Console.WriteLine($"Admin: {Nama}");
}
Analogi: role baru mewarisi aturan dasar komunitas, lalu nambah kemampuan khusus.

E.10 Object Etiquette

Object etiquette itu etika desain object: enkapsulasi, naming jelas, validasi input, dan antarmuka yang konsisten.

// Etika object: private field, validasi di setter, method jelas
class SaldoAkun
{
    private double _saldo;

    public double Saldo => _saldo; // read-only dari luar

    public SaldoAkun(double saldoAwal)
    {
        if (saldoAwal < 0) throw new ArgumentException("Saldo awal tidak boleh negatif.");
        _saldo = saldoAwal;
    }

    public void Deposit(double jumlah)
    {
        if (jumlah <= 0) throw new ArgumentException("Jumlah deposit harus positif.");
        _saldo += jumlah;
        Console.WriteLine($"Deposit berhasil. Saldo sekarang: {_saldo:C}");
    }
}

var akun = new SaldoAkun(100000);
akun.Deposit(50000);
Console.WriteLine($"Saldo final: {akun.Saldo:C}");
Analogi: etika kerja tim bikin komunikasi rapi dan minim drama.

E.11 The Power of Strings and Chars

String dan char penting banget untuk parsing input, validasi format, dan manipulasi teks.

E.13 Building a Bank

Bab ini menyatukan semuanya: object modeling, validasi, error handling, dan desain komponen jadi satu solusi utuh.

// Integrasi: enum + class + property + validasi + method
enum StatusTransaksi { Berhasil, Gagal }

class BankAccount
{
    public string Nama { get; }
    private double _saldo;

    public BankAccount(string nama, double saldoAwal)
    {
        Nama = nama;
        _saldo = saldoAwal;
    }

    public StatusTransaksi Tarik(double jumlah)
    {
        if (jumlah <= 0 || jumlah > _saldo)
            return StatusTransaksi.Gagal;

        _saldo -= jumlah;
        return StatusTransaksi.Berhasil;
    }

    public void Info()
        => Console.WriteLine($"[{Nama}] Saldo: {_saldo:C}");
}

var acc = new BankAccount("Dila", 500000);
var hasil = acc.Tarik(200000);
Console.WriteLine($"Status penarikan: {hasil}");
acc.Info();
Analogi: ini momen dari latihan drill ke pertandingan beneran.

BAB F. Advanced Programming

F.1 Generics and Collections

Generics bikin collection lebih type-safe dan reusable tanpa casting aneh-aneh.

List<string> namaUser = new List<string> { "Ari", "Bela" };

Analogi: generics itu wadah yang jelas label isinya, jadi gak campur aduk.

F.2 Storing Business Objects

Business object perlu disimpan dengan strategi yang terstruktur: file, database, atau service.

using System.Text.Json;

    class Customer
    {
      public string Id { get; set; }
      public string Nama { get; set; }
    }

    var customers = new List<Customer>
    {
      new Customer { Id = "C001", Nama = "Ari" },
      new Customer { Id = "C002", Nama = "Bela" }
    };

    string json = JsonSerializer.Serialize(customers);
    File.WriteAllText("customers.json", json);
    Console.WriteLine("Data customer tersimpan ke customers.json");
Analogi: data pelanggan gak cukup diingat kepala, harus dicatat rapi biar konsisten.

F.3 Business Objects and Editing

Edit data objek butuh validasi dan audit agar perubahan tidak merusak integritas data.

class ProdukBisnis
    {
      public string Nama { get; private set; }
      public decimal Harga { get; private set; }

      public ProdukBisnis(string nama, decimal harga)
      {
        Nama = nama;
        Harga = harga;
      }

      public void UpdateHarga(decimal hargaBaru)
      {
        if (hargaBaru <= 0)
          throw new ArgumentException("Harga harus lebih dari 0.");

        Console.WriteLine($"Audit: harga {Nama} berubah dari {Harga:C} ke {hargaBaru:C}");
        Harga = hargaBaru;
      }
    }

    var kopi = new ProdukBisnis("Kopi Susu", 18000);
    kopi.UpdateHarga(20000);
Analogi: edit daftar harga harus jelas siapa yang ubah dan kapan.

F.6 Program Organisation

Program organization mencakup struktur folder, pemisahan layer, naming convention, dan dependency management.

// Simulasi layer sederhana: Service bergantung ke Repository interface
    interface IOrderRepository
    {
      void Save(string orderId);
    }

    class OrderRepository : IOrderRepository
    {
      public void Save(string orderId) => Console.WriteLine($"Order {orderId} disimpan");
    }

    class OrderService
    {
      private readonly IOrderRepository _repo;

      public OrderService(IOrderRepository repo)
      {
        _repo = repo;
      }

      public void CreateOrder(string orderId)
      {
        _repo.Save(orderId);
      }
    }

    new OrderService(new OrderRepository()).CreateOrder("ORD-1001");
Analogi: gudang rapi bikin barang gampang dicari dan operasional gak macet.

F.8 Debugging

Debugging adalah skill inti untuk cari akar masalah, bukan sekadar menutup gejala.

  1. Reproduce bug.
  2. Isolasi sumber masalah.
  3. Perbaiki akar masalah.
  4. Retest alur terkait.
int[] angka = { 10, 20, 30 };
int index = 3;

try
{
    Console.WriteLine($"Nilai: {angka[index]}");
}
catch (IndexOutOfRangeException ex)
{
    Console.WriteLine($"Debug: index {index} di luar batas array ({angka.Length}).");
    Console.WriteLine(ex.Message);
}
Analogi: speaker rusak gak cukup dikecilin volumenya, harus ketemu sumber noise aslinya.

BAB G. Glossary of Terms

G.1 Istilah OOP dan Struktur Program

Istilah inti yang wajib lo pegang: class, object, method, property, constructor, inheritance, interface, static, override, overload.

  • Class: blueprint.
  • Object: instance dari class.
  • Property: akses data yang terkontrol.
  • Method: aksi dari object.
Analogi: class itu resep, object itu makanan yang jadi.

G.2 Istilah Desain dan Arsitektur

Istilah penting: cohesion, coupling, component, dependency, code reuse, hierarchy, unit test, test harness, namespace.

  • Cohesion tinggi: satu modul fokus satu tujuan.
  • Coupling rendah: antar modul tidak saling nempel berlebihan.
  • Dependency: ketergantungan antar bagian.
Analogi: tim solid itu tugasnya jelas (cohesion) dan gak saling ngerepotin (low coupling).

G.3 Istilah Runtime, Data, dan Akses

Istilah pendukung: compiler, machine code, metadata, stream, exception, GUID, immutable, value type, reference type, public/private/protected.

// akses modifier mini contoh
public class UserService
{
    private string _token = "secret";
    public string NamaAplikasi { get; set; } = "CSharpApp";
}
Analogi: access modifier itu kayak level akses ruang kerja, gak semua orang boleh masuk semua area.
