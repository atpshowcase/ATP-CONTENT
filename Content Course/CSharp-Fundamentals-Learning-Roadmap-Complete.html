BAB A. Introduction to CSharp

A.1 What is CSharp

CSharp adalah bahasa pemrograman modern dari Microsoft yang jalan di atas platform .NET. Bahasa ini kuat buat bikin aplikasi bisnis, API, desktop app, sampai game.

Kenapa banyak dipakai? Karena sintaksnya rapi, tooling-nya matang, dan ekosistem library-nya luas banget buat kebutuhan real project.

// Program C# paling basic
Console.WriteLine("Halo CSharp");
Analogi: CSharp itu kayak kendaraan serbaguna. Mau dipakai ngangkut barang, antar orang, atau perjalanan jauh, semua ada paket fiturnya.

A.2 History and Ecosystem of .NET

.NET dulu identik dengan Windows, tapi sekarang sudah cross-platform. Lo bisa develop dan deploy di Windows, Linux, dan macOS.

  • .NET SDK buat build dan run aplikasi.
  • NuGet buat dependency management.
  • ASP.NET Core buat web development.
dotnet --info

dotnet --list-sdks
Analogi: dulu tokonya cuma buka di satu kota, sekarang sudah buka cabang di banyak kota dengan standar layanan yang sama.

A.3 Where CSharp is Used

Di dunia kerja, CSharp dipakai di banyak area:

  • Web backend dan Web API (ASP.NET Core).
  • Desktop app internal perusahaan.
  • Game development pakai Unity.
  • Background service, worker, dan automation script.
// contoh endpoint minimal API
app.MapGet("/ping", () => "pong");
Analogi: satu skill yang bisa dipakai di banyak medan, dari warung kecil sampai pabrik besar.

A.4 Installing Tools

Setup minimal biar langsung ngoding:

  1. Install .NET SDK versi stabil terbaru.
  2. Pilih editor: Visual Studio atau VS Code.
  3. Test buat project baru dan jalankan.
dotnet new console -n HelloCSharp
cd HelloCSharp
dotnet run
Analogi: sebelum masak, pastikan kompor, gas, dan alat dapur semua siap dulu.

BAB B. Basic Syntax and Structure

B.1 Structure of a CSharp Program

Program CSharp modern bisa super singkat (top-level statements), tapi konsep dasarnya tetap sama: ada namespace, class (opsional di template modern), dan entry point aplikasi.

using System;

Console.WriteLine("Program dimulai");
Analogi: rumah bisa model minimalis, tapi tetap ada pintu masuk, ruang utama, dan jalur listrik.

B.2 Main Method and Program Entry

Di versi klasik, aplikasi console mulai dari method Main(). Ini titik pertama yang dieksekusi runtime.

class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("Masuk Main");
    }
}
Analogi: Main itu pintu gerbang. Semua alur masuk dari sana dulu.

B.3 Namespaces and Comments

Namespace dipakai buat ngelompokin kode biar gak tabrakan nama. Comment dipakai buat kasih konteks penting, bukan ngulang hal yang sudah jelas.

namespace BelajarDasar;

// Ini komentar satu baris
/*
   Ini komentar multi baris
*/
Analogi: namespace itu kayak alamat RT/RW. Nama orang bisa sama, tapi alamat beda bikin tetap jelas.

B.4 Basic Input and Output

Input output dasar di console dipakai buat latihan logic awal.

Console.Write("Nama kamu: ");
string nama = Console.ReadLine() ?? "Anon";
Console.WriteLine($"Halo, {nama}");
Analogi: ini kayak kasir yang nanya nama pelanggan lalu panggil sesuai data yang masuk.

BAB C. Variables and Basic Logic

C.1 Variables and Data Types

Variabel itu tempat nyimpen data. Di CSharp ada value type dan reference type.

  • Value type: int, double, bool, char.
  • Reference type: string, object, class.
int umur = 24;
double tinggi = 172.5;
bool aktif = true;
char grade = 'A';
string nama = "Raka";
Analogi: value type itu kayak uang tunai di dompet sendiri, reference type kayak kartu akses ke loker.

C.2 Type Casting and var Keyword

Cast dipakai saat lo pindah tipe data. Implicit casting aman kalau gak ada risiko kehilangan data, explicit casting harus sadar konsekuensinya.

int angka = 10;
double implicitCast = angka;

double harga = 12.9;
int explicitCast = (int)harga;

var kota = "Jakarta"; // compiler infer string
Analogi: pindah barang dari truk besar ke motor perlu sortir dulu, gak semua barang bisa dibawa utuh.

C.3 Operators

Operator dipakai buat ngitung, ngebandingin, dan nyusun logika keputusan.

int a = 8;
int b = 3;

int total = a + b;
bool lebihBesar = a > b;
bool valid = (a > 0) && (b > 0);
a++;
Analogi: operator itu alat kerja di bengkel, ada kunci buat buka baut, ada alat ukur buat cek hasil.

C.4 Control Flow

Control flow ngatur arah program berdasarkan kondisi.

int nilai = 78;

if (nilai >= 85)
{
    Console.WriteLine("A");
}
else if (nilai >= 70)
{
    Console.WriteLine("B");
}
else
{
    Console.WriteLine("Perlu latihan lagi");
}
Analogi: satpam parkir ngarahin kendaraan berdasarkan kondisi lahan yang lagi tersedia.

C.5 Looping

Loop dipakai saat lo perlu ngulang aksi berkali-kali tanpa nulis kode berulang.

for (int i = 1; i <= 3; i++)
{
    Console.WriteLine($"Loop for ke-{i}");
}

int n = 0;
while (n < 2)
{
    Console.WriteLine("Loop while");
    n++;
}
Analogi: kayak barista bikin 20 gelas kopi, prosesnya sama tapi jumlahnya banyak.

BAB D. Methods and Collections

D.1 Declaring Methods and Return Types

Method bikin kode lebih modular dan gampang diuji. Kalau gak butuh nilai balik pakai void, kalau butuh hasil pakai return type.

static void Sapa(string nama)
{
    Console.WriteLine($"Halo {nama}");
}

static int Tambah(int x, int y)
{
    return x + y;
}
Analogi: method itu SOP kerja. Tinggal panggil prosedurnya, hasilnya konsisten.

D.2 Overloading and Optional Parameters

Overloading bikin satu nama method bisa punya beberapa variasi parameter. Optional dan named parameter bikin pemanggilan lebih fleksibel.

static int Hitung(int a, int b) => a + b;
static int Hitung(int a, int b, int c) => a + b + c;

static void Cetak(string teks, int repeat = 1)
{
    for (int i = 0; i < repeat; i++)
    {
        Console.WriteLine(teks);
    }
}

Cetak(teks: "Belajar CSharp", repeat: 2);
Analogi: menu paket bisa ukuran kecil atau besar, nama menu sama tapi isi beda.

D.3 Arrays and Multidimensional Arrays

Array cocok saat ukuran data relatif tetap. Buat data tabel, bisa pakai multidimensional array.

int[] angka = { 10, 20, 30 };
int[,] matrix =
{
    { 1, 2 },
    { 3, 4 }
};

Console.WriteLine(matrix[1, 0]); // 3
Analogi: array itu rak dengan slot bernomor, jadi lo ambil barang berdasarkan nomor slot.

D.4 List and Dictionary

List<T> dipakai kalau jumlah data dinamis. Dictionary<TKey, TValue> dipakai kalau lo butuh lookup cepat berdasarkan key.

var daftarNama = new List<string> { "Ayu", "Bima" };
daftarNama.Add("Citra");

var skor = new Dictionary<string, int>
{
    ["Ayu"] = 90,
    ["Bima"] = 85
};

Console.WriteLine(skor["Ayu"]);
Analogi: list itu antrean orang, dictionary itu buku kontak dengan nama sebagai kunci.

BAB E. OOP and Design Foundation

E.1 Classes, Objects, Properties, Fields, Constructors

Class adalah cetak biru, object adalah instansinya. Field biasanya dipakai internal class, property dipakai sebagai akses terkontrol dari luar.

class Mahasiswa
{
    private string _nim;
    public string Nama { get; set; }

    public Mahasiswa(string nim, string nama)
    {
        _nim = nim;
        Nama = nama;
    }
}
Analogi: class itu blueprint rumah, object itu rumah yang benar-benar dibangun.

E.2 Encapsulation

Encapsulation artinya lo sembunyikan detail internal dan kasih akses via antarmuka yang aman.

class Dompet
{
    private decimal _saldo;

    public void TopUp(decimal nominal)
    {
        if (nominal <= 0) return;
        _saldo += nominal;
    }

    public decimal LihatSaldo() => _saldo;
}
Analogi: kasir gak kasih lo pegang laci uang langsung, tapi kasih jalur resmi buat transaksi.

E.3 Inheritance, Polymorphism, Abstraction

Inheritance buat nurunin behavior, polymorphism buat perlakuan berbeda dari kontrak sama, abstraction buat fokus ke hal penting tanpa expose detail rumit.

abstract class Hewan
{
    public abstract void Suara();
}

class Kucing : Hewan
{
    public override void Suara() => Console.WriteLine("Meong");
}

Hewan h = new Kucing();
h.Suara();
Analogi: semua kendaraan punya method "jalan", tapi cara jalannya beda antara motor, mobil, dan kereta.

E.4 Interfaces and Abstract Classes

Interface cocok buat definisi kontrak. Abstract class cocok saat lo mau share sebagian implementasi dasar.

interface IPembayaran
{
    void Bayar(decimal nominal);
}

class EWallet : IPembayaran
{
    public void Bayar(decimal nominal)
    {
        Console.WriteLine($"Bayar {nominal}");
    }
}

Di level desain dependency, biasakan bergantung ke interface, bukan implementasi konkret, supaya kode gampang diganti dan di-test.

Analogi: colokan listrik itu interface. Mau charger merek apa pun, selama plug-nya cocok, tetap bisa dipakai.

BAB F. Error Handling and Data Processing

F.1 Exception Handling Basics

Gunakan try-catch-finally untuk ngamanin operasi yang berpotensi gagal, terutama I/O dan external dependency.

try
{
    int hasil = 10 / int.Parse("0");
    Console.WriteLine(hasil);
}
catch (DivideByZeroException ex)
{
    Console.WriteLine($"Error: {ex.Message}");
}
finally
{
    Console.WriteLine("Selesai proses");
}
Analogi: lo tetap gas, tapi pakai helm dan rem yang siap dipakai saat kondisi bahaya.

F.2 Custom Exceptions and Best Practices

Custom exception berguna saat domain lo butuh error yang lebih meaningful. Hindari catch semua exception tanpa aksi yang jelas.

class SaldoTidakCukupException : Exception
{
    public SaldoTidakCukupException(string message) : base(message) { }
}

throw new SaldoTidakCukupException("Saldo tidak cukup untuk transaksi.");
  • Throw exception dengan pesan yang jelas.
  • Log context penting sebelum rethrow.
  • Jangan pakai exception untuk flow normal.
Analogi: alarm kebakaran dipakai saat beneran darurat, bukan buat ngasih tahu jam makan siang.

F.3 File Handling and Streams

Untuk baca tulis file sederhana, class File sudah cukup. Kalau proses data besar, pakai stream supaya lebih efisien memory.

string path = "catatan.txt";
File.WriteAllText(path, "Belajar CSharp step by step");

string isi = File.ReadAllText(path);
Console.WriteLine(isi);
Analogi: ember kecil cukup buat satu ember air, tapi buat isi kolam lo butuh aliran pipa yang stabil.

F.4 LINQ Fundamentals

LINQ bikin query data di CSharp jadi lebih ekspresif dan gampang dibaca.

var angka = new List<int> { 2, 5, 8, 11, 14 };

var genap = angka.Where(x => x % 2 == 0);
var kuadrat = angka.Select(x => x * x);
var urutDesc = angka.OrderByDescending(x => x);

Console.WriteLine(string.Join(", ", genap));
Analogi: LINQ itu kayak lo punya filter pencarian di marketplace, tinggal pilih syarat lalu data langsung rapih.

BAB G. Async and Project Structure

G.1 Async and Await Basics

Async dipakai biar aplikasi gak nge-freeze saat nunggu proses lambat seperti API call atau file I/O.

static async Task<string> AmbilDataAsync()
{
    await Task.Delay(500);
    return "Data siap";
}

string hasil = await AmbilDataAsync();
Console.WriteLine(hasil);

Pakai async secara sadar: untuk operasi I/O bound, bukan buat semua hal secara membabi buta.

Analogi: sambil nunggu pesanan dicetak, kasir tetap bisa layani pelanggan lain.

G.2 Understanding Project Files and Build Run Process

File .csproj adalah pusat konfigurasi project: target framework, package, dan setting build. Paham ini bikin lo lebih pede saat setup project tim.

dotnet new console -n BelajarProject
cd BelajarProject

dotnet build
dotnet run
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net8.0</TargetFramework>
  </PropertyGroup>
</Project>
Analogi: .csproj itu checklist dapur produksi. Bahan, alat, dan standar kerjanya ditulis di satu tempat.

BAB H. Web Direction and Next Steps

H.1 ASP.NET Core Basics

Setelah fondasi CSharp kuat, langkah natural berikutnya adalah ASP.NET Core untuk bikin backend web modern.

dotnet new webapi -n BelajarApi
cd BelajarApi
dotnet run
Analogi: habis jago masak di dapur rumah, sekarang lo buka dapur komersial yang melayani banyak pelanggan.

H.2 Web API Fundamentals

Web API adalah cara aplikasi saling ngobrol lewat HTTP. Endpoint umum: GET, POST, PUT, DELETE.

app.MapGet("/products", () => new[] { "Keyboard", "Mouse" });
app.MapPost("/products", (string name) => Results.Ok(name));
Analogi: API itu kasir restoran online. Pelanggan kirim order, dapur proses, hasil dikirim balik.

H.3 MVC Pattern High Level

MVC ngebagi tanggung jawab jadi:

  • Model untuk data dan aturan.
  • View untuk tampilan.
  • Controller untuk penghubung alur request.
// Controller nerima request
// Model pegang data
// View render hasil
Analogi: di restoran, kasir terima pesanan, dapur olah makanan, waiter antar ke meja pelanggan.

H.4 Learning Path After Fundamentals

Biar skill lo naik stabil, ini urutan lanjut yang realistis:

  1. Bangun 1-2 project CRUD sampai deploy.
  2. Masuk ke Entity Framework Core untuk akses database.
  3. Belajar authentication authorization (JWT, roles, policy).
  4. Pahami testing (unit test, integration test) dan logging.
  5. Lanjut ke clean architecture dan system design dasar.
Analogi: bukan sprint sekali jadi. Ini seperti latihan gym, progres paling kebentuk dari konsistensi mingguan.
Kesimpulan: Kalau pondasi CSharp lo kuat dari syntax sampai OOP, LINQ, dan async, proses naik ke ASP.NET Core bakal jauh lebih mulus. Fokus ke praktik bertahap, bukan buru-buru ngejar framework tanpa dasar.
