Tiptap Style Notes - A sampai F .NET Learning Path

Fundamental .NET Sampai Praktik Aplikasi

Bahas santai tapi runut, mulai dari pondasi .NET, lanjut struktur project, Web API, sampai best practice dunia nyata.

A. Fundamental .NET

A.1 Pengenalan Platform .NET

.NET itu platform pengembangan aplikasi dari Microsoft yang bisa dipakai buat bikin web, desktop, mobile, cloud service, sampai background worker. Jadi bukan cuma bahasa C#, tapi satu ekosistem lengkap: runtime, library, tooling, dan CLI.

  • Runtime: mesin yang ngejalanin aplikasi lo.
  • Base Class Library (BCL): kumpulan fungsi bawaan biar gak coding dari nol.
  • SDK dan CLI: alat buat build, run, test, publish.
  • Bahasa utama: C#, F#, dan Visual Basic.
// cek SDK .NET yang terpasang
// dotnet --info

// bikin project console baru
// dotnet new console -n BelajarDotNet

// jalanin project
// dotnet run
Analogi: .NET itu kayak satu komplek olahraga lengkap. Lo bisa main futsal, basket, gym di area yang sama karena fasilitasnya udah disatuin.

A.2 Arsitektur Dasar .NET

Arsitektur dasar .NET bisa lo lihat sebagai alur berlapis: kode C# lo di-compile jadi IL, lalu dieksekusi CLR. CLR ini yang ngurus memory management, garbage collection, type safety, dan keamanan runtime.

  1. Source Code (C#): kode yang lo tulis.
  2. Compiler (Roslyn): ngubah kode jadi Intermediate Language (IL).
  3. CLR (Common Language Runtime): ngejalanin IL, optimasi via JIT, ngelola memori.
  4. BCL/NuGet Libraries: fungsi tambahan untuk networking, file, JSON, database, dan lain-lain.
// Alur sederhananya:
// C# code -> IL -> CLR/JIT -> Machine Code

string nama = "Platform .NET";
Console.WriteLine($"Belajar {nama} dari arsitektur dasarnya");
Analogi: ini kayak dapur restoran. Lo (developer) nulis pesanan, chef (compiler + CLR) yang olah jadi makanan siap saji buat mesin.

A.3 Perbedaan .NET Framework, .NET Core, dan .NET Modern

Biar gampang, anggap evolusi .NET itu ada tiga fase utama. Sekarang yang direkomendasikan buat project baru adalah .NET Modern (versi 6/7/8 ke atas).

  • .NET Framework: generasi lama, dominan Windows, cocok untuk legacy enterprise lama.
  • .NET Core: generasi transisi yang cross-platform (Windows, Linux, macOS), performa lebih ringan.
  • .NET Modern (nama produknya balik jadi .NET): penerus resmi .NET Core dengan LTS/STS, tooling makin matang, ekosistem paling aktif.
// target framework di file .csproj
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net8.0</TargetFramework>
  </PropertyGroup>
</Project>
Analogi: .NET Framework itu motor lama yang masih bisa jalan, .NET Core itu versi upgrade awal, dan .NET Modern itu model terbaru yang paling irit, kenceng, dan paling banyak support bengkelnya.

A.4 Sejarah .NET

Sejarah .NET dimulai awal 2000-an waktu Microsoft ngenalin .NET Framework (2002) buat nyatuin cara bikin aplikasi Windows dan web. Setelah itu, ekosistemnya berkembang cepat sampai akhirnya lahir .NET Core (2016) yang cross-platform dan jadi fondasi .NET modern.

  • 2002: .NET Framework 1.0 resmi rilis.
  • 2016: .NET Core 1.0 rilis, mulai era cross-platform.
  • 2020: .NET 5 nyatuin branding jadi satu jalur.
  • 2021-sekarang: .NET 6/7/8 fokus performa, cloud, dan produktivitas.
// Timeline versi utama
string[] timeline =
{
    "2002 - .NET Framework 1.0",
    "2016 - .NET Core 1.0",
    "2020 - .NET 5",
    "2021 - .NET 6 (LTS)",
    "2022 - .NET 7",

foreach (var item in timeline)

    Console.WriteLine(item);
}
Analogi: perjalanan .NET itu kayak evolusi kendaraan operasional dari model lama ke model hybrid modern, mesinnya makin efisien tapi tujuan tetap sama: nganter kerjaan lo sampai tujuan.

B. Framework Web pada .NET

B.1 Pengenalan ASP.NET

ASP.NET adalah framework web di ekosistem .NET buat bikin aplikasi web dinamis, API, dan real-time app. Intinya, kalau lo mau bangun backend web di .NET, titik masuk paling umum ya ASP.NET.

  • Dukungan routing HTTP (GET, POST, PUT, DELETE).
  • Built-in middleware pipeline.
  • Integrasi kuat sama authentication, logging, dan DI.
  • Cocok untuk monolith maupun microservices.
// Minimal API (ASP.NET Core)
var builder = WebApplication.CreateBuilder(args);
var app = builder.Build();

app.MapGet("/", () => "Halo dari ASP.NET!");

app.Run();
Analogi: ASP.NET itu kayak dapur utama restoran online, semua order web masuk lewat sini lalu diproses rapi.

B.2 ASP.NET MVC

ASP.NET MVC pakai pola Model-View-Controller biar code terpisah per tanggung jawab: data, tampilan, dan alur request. Pola ini enak banget buat aplikasi web yang server-rendered dan butuh struktur tim yang jelas.

  1. Model: representasi data dan aturan bisnis.
  2. View: tampilan HTML yang dilihat user.
  3. Controller: jembatan request user ke logic aplikasi.
// Controller sederhana di ASP.NET MVC
public class HomeController : Controller
{
    public IActionResult Index()
    {
        ViewData["Message"] = "Selamat datang di ASP.NET MVC";
        return View();
    }
}
Analogi: MVC itu kayak tim produksi konten: penulis (Model), editor layout (View), sama produser yang ngatur alur kerja (Controller).

B.3 ASP.NET Core

ASP.NET Core adalah versi modern yang lebih ringan, cepat, dan cross-platform. Ini framework web utama yang dipakai sekarang untuk project baru karena performa tinggi dan deployment fleksibel ke Linux, Windows, container, sampai cloud.

  • Cross-platform: jalan di Windows, Linux, macOS.
  • Performa tinggi dan memory footprint lebih efisien.
  • Built-in dependency injection.
  • Mendukung MVC, Razor Pages, Web API, SignalR, Minimal API.
// Registrasi service + endpoint di ASP.NET Core
var builder = WebApplication.CreateBuilder(args);

builder.Services.AddControllers();

var app = builder.Build();
app.MapControllers();
app.Run();
Analogi: ASP.NET Core itu kayak kendaraan operasional terbaru: lebih irit, lebih cepat, dan bisa dipakai di lebih banyak medan.

C. Struktur Project dan Komponen

Struktur Project .NET

Project .NET itu biasanya udah punya pola folder, jadi tim lo gampang baca alur code walau orangnya beda-beda.

  • Controllers: pintu masuk request user.
  • Models: representasi data atau entitas.
  • Views: tampilan yang diliat user (kalau project MVC).
  • File konfigurasi: ngatur environment, dependency, dan behavior app.

Konsep Layer Architecture

Layer architecture itu teknik biar tanggung jawab code gak campur aduk.

  1. Controller Layer: nerima request dan ngarahin proses.
  2. Service Layer: isi business logic utama.
  3. Repository Layer: urusan akses data/database.
  4. Domain Model: bentuk data inti sesuai kebutuhan bisnis.

Dependency dan Package Management

Di .NET, urusan package biasanya lewat NuGet. Mirip package manager di ekosistem lain, tapi khusus dunia .NET.

  • NuGet dipakai buat install library pihak ketiga.
  • Versi package perlu dijaga biar gak bentrok dependency.
  • Penambahan package bisa via CLI atau Visual Studio.

D. Dasar Pengembangan Web API

Konsep Web API

API itu jembatan komunikasi antar aplikasi. Frontend minta data, backend ngasih response.

  • REST API di .NET biasanya pakai endpoint berbasis HTTP method (GET, POST, PUT, DELETE).
  • Format data paling umum: JSON.
  • API harus konsisten biar gampang dipakai tim frontend/mobile.

Membuat API Sederhana

Start simple dulu biar alur kebaca.

  1. Bikin controller API.
  2. Set routing dasar endpoint.
  3. Balikin response JSON dari action method.
Tips tongkrongan: jangan keburu mikir kompleks. API sederhana yang rapi lebih berguna daripada API canggih tapi bikin tim bingung.

E. Praktik Pengembangan Aplikasi

Membuat Aplikasi Web Sederhana

Di tahap praktik, fokusnya bukan keren-kerenan fitur, tapi ngebentuk alur kerja yang bener.

  • Bikin project baru sesuai template yang cocok.
  • Tambah controller buat nangani request.
  • Tampilkan data ke view biar alur end-to-end keliatan.

Menghubungkan Database

Kalau aplikasi lo cuma nyimpen data di variabel, begitu di-restart ya ilang. Biar data bener-bener nyata dan persistent, lo harus nyambungin ke database. Di sini kita masuk ke topik yang krusial banget.

Konsep ORM

ORM itu singkatan dari Object-Relational Mapping. Intinya: lo nulis kode pakai object (kayak class Product, User, dll), terus si ORM yang ngurusin gimana data itu disimpen dan dibaca dari tabel database. Lo gak perlu nulis SQL raw buat tiap operasi.

Bayangin lo punya class kayak gini:

public class Product
{
    public int Id { get; set; }
    public string Name { get; set; }
}

Dengan ORM, class ini otomatis ke-map ke tabel Products di database. Field Id jadi kolom Id, Name jadi kolom Name. Lo gak perlu nulis SQL sama sekali.

Keuntungannya? Lo nulis kode dalam bahasa C#, bukan SQL. Lebih readable, lebih type-safe, dan lebih gampang di-maintain.

Pengenalan Entity Framework

Entity Framework (EF) adalah ORM bawaan Microsoft buat .NET. Ini yang paling umum dipake di dunia .NET, dan lo pasti bakal ketemu ini di hampir semua project.

Ada dua pendekatan utama di EF:

  • Code First — lo bikin class C# dulu, terus EF generate tabel databasenya. Cocok buat project baru.
  • Database First — lo udah punya database, terus EF generate class C#-nya dari tabel yang ada. Cocok buat project lama.

Yang paling sering dipake sekarang itu Code First. Lo bikin model, bikin DbContext (ini semacam "pintu masuk" ke database lo), terus jalanin migration buat generate tabelnya.

Flow-nya simpel: Bikin Model → Bikin DbContext → Tambah DbSet → Jalanin Migration → Tabel terbentuk. Selesai. Database lo udah siap dipake.

CRUD Dasar

CRUD itu empat operasi paling basic yang lo lakuin ke database: Create, Read, Update, Delete. Hampir semua fitur aplikasi berputar di empat hal ini.

// CREATE — tambahin data baru
var product = new Product { Name = "Laptop" };
db.Products.Add(product);
db.SaveChanges();

// READ — ambil semua data, atau cari satu berdasarkan ID
var semua = db.Products.ToList();
var satu  = db.Products.Find(1);

// UPDATE — ambil dulu, ubah, terus save
var p = db.Products.Find(1);
p.Name = "Laptop Pro";
db.SaveChanges();

// DELETE — ambil dulu, hapus, terus save
var p = db.Products.Find(1);
db.Products.Remove(p);
db.SaveChanges();

Lo bakal notice satu pattern yang sama terus: ambil dari context → modifikasi → save. Itu inti dari cara kerja EF.

Tips: Jangan skip latihan CRUD manual dulu. Banyak yang langsung lompat ke scaffold auto-generate, tapi kalau lo gak ngerti gimana EF kerja di baliknya, lo bakal bingung waktu ada bug atau mau optimasi query.

F. Penutup

Best Practice Pengembangan .NET

Pas project makin gede, best practice itu bukan opsional lagi, tapi wajib.

  • Jaga struktur project biar tim gampang maintain.
  • Tulis clean code biar logic gampang dibaca dan dites.
  • Pakai modular architecture supaya scaling lebih enak.
Ringkasnya: kuatin fondasi dulu, baru ngebut ke fitur. Di .NET, arsitektur rapi dari awal itu ngasih dampak gede buat jangka panjang.
