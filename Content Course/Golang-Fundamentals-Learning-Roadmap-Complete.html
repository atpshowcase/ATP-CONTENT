BAB A. Introduction and Setup

A.1 What is Go and Why It Matters

Go adalah bahasa pemrograman yang didesain buat sederhana, cepat, dan gampang dipelihara. Filosofinya jelas: sintaks jangan ribet, performa tetap kencang, dan concurrency jadi warga kelas satu.

Di dunia kerja, Go sering dipilih buat backend service, microservices, CLI tool, dan sistem cloud karena build-nya cepat dan deployment-nya enak.

package main

import "fmt"

func main() {
    fmt.Println("Halo Go")
}
Analogi: Go itu kayak motor harian yang sederhana, irit, tapi tetap kuat buat perjalanan jauh.

A.2 History and Philosophy of Go

Go lahir di Google untuk ngatasin problem software skala besar: waktu compile lama, dependency ribet, dan maintenance codebase yang berat.

  • Simplicity: style coding dibuat konsisten.
  • Performance: compiled language, native binary.
  • Concurrency: goroutines dan channels built-in.
go version

go env
Analogi: dibanding pisau Swiss army yang serba ada, Go lebih kayak pisau dapur utama yang fokus, tajam, dan dipakai tiap hari.

A.3 Installing Go and Environment Setup

Setup minimum: install Go, pastikan PATH benar, lalu test program pertama. Untuk project modern, pakai module dari awal biar dependency rapi.

go version

mkdir belajar-go
cd belajar-go
go mod init belajar-go

go run main.go
Analogi: sebelum masak, lo cek kompor nyala, gas aman, dan bahan sudah siap di meja.

BAB B. Basic Syntax and Data

B.1 Program Structure in Go

Setiap program executable Go dimulai dari package main dan func main(). Import dipakai buat ngambil package standar atau package sendiri.

package main

import "fmt"

func main() {
    fmt.Println("Program dimulai")
}
Analogi: package main itu pintu masuk venue, func main itu titik mulai acaranya.

B.2 Variables and Basic Data Types

Di Go, lo bisa deklarasi variable pakai var atau shorthand :=. Tipe dasar yang paling sering dipakai: int, float64, bool, string.

var umur int = 25
kota := "Bandung"
aktif := true
nilai := 87.5

fmt.Println(umur, kota, aktif, nilai)
Analogi: variable itu laci-laci penyimpanan, setiap laci punya label isi yang jelas.

B.3 Zero Values and Constants

Kalau variable dideklarasi tanpa nilai awal, Go kasih zero value otomatis. Ini bikin behavior default lebih predictable.

  • int jadi 0
  • bool jadi false
  • string jadi ""
var total int
var isReady bool
var nama string

const appName = "BelajarGo"

fmt.Println(total, isReady, nama, appName)
Analogi: kalau lo belum isi form, sistem tetap kasih nilai default biar alur gak rusak.

B.4 Type Conversion and Operators

Go tegas soal tipe data, jadi conversion harus eksplisit. Operator standar dipakai buat hitung, bandingin, dan bikin logika.

a := 10
b := 3

hasil := a + b
isBesar := a > b
valid := (a > 0) && (b > 0)

rata := float64(a) / float64(b)

fmt.Println(hasil, isBesar, valid, rata)
Analogi: barang beda ukuran harus disesuaikan dulu sebelum masuk rak yang sama.

BAB C. Control Flow and Functions

C.1 If Else and Switch

Control flow dipakai buat ngatur jalur program berdasarkan kondisi. Go juga punya short statement di if yang sering kepake buat validasi cepat.

nilai := 82

if nilai >= 85 {
    fmt.Println("Grade A")
} else if nilai >= 70 {
    fmt.Println("Grade B")
} else {
    fmt.Println("Perlu latihan")
}

switch {
case nilai >= 90:
    fmt.Println("Excellent")
case nilai >= 75:
    fmt.Println("Good")
default:
    fmt.Println("Keep learning")
}
Analogi: kayak satpam parkir yang ngarahin kendaraan ke jalur beda sesuai kondisi lapangan.

C.2 Looping with For

Go cuma punya satu keyword loop: for. Tapi bentuknya fleksibel, bisa gaya klasik, mirip while, sampai infinite loop.

for i := 1; i <= 3; i++ {
    fmt.Println("for klasik", i)
}

n := 0
for n < 2 {
    fmt.Println("for ala while", n)
    n++
}

for {
    fmt.Println("infinite loop")
    break
}
Analogi: satu remote bisa buat banyak fungsi, tinggal mode pemakaiannya yang beda.

C.3 Functions and Multiple Return Values

Function di Go bisa ngembaliin lebih dari satu nilai. Ini fitur inti yang sering dipakai bareng error handling.

func bagi(a int, b int) (int, int) {
    return a / b, a % b
}

hasil, sisa := bagi(10, 3)
fmt.Println(hasil, sisa)
Analogi: sekali transaksi bisa dapet struk dan kembalian sekaligus.

C.4 Variadic and Anonymous Functions

Variadic function menerima jumlah argumen yang fleksibel. Anonymous function cocok buat operasi cepat atau callback kecil.

func jumlah(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

double := func(x int) int {
    return x * 2
}

fmt.Println(jumlah(1, 2, 3, 4))
fmt.Println(double(5))
Analogi: variadic itu kayak lo buka meja makan buat tamu jumlahnya bisa berubah-ubah.

BAB D. Arrays Slices Maps and Structs

D.1 Arrays and Slices Core Concept

Array punya ukuran tetap, slice lebih fleksibel dan dipakai hampir di semua project Go. Memahami slice itu wajib kalau mau nyaman nulis Go.

arr := [3]int{10, 20, 30}
sl := []int{1, 2, 3}

sl = append(sl, 4)

fmt.Println(arr)
fmt.Println(sl)
fmt.Println(len(sl), cap(sl))
Analogi: array itu rak permanen, slice itu keranjang belanja yang bisa nambah isi.

D.2 Slice Operations and Range Iteration

Lo bisa ambil bagian slice, iterasi pakai range, dan manfaatin len/cap buat baca kondisi data saat runtime.

data := []string{"go", "python", "java", "rust"}

subset := data[1:3]
fmt.Println(subset)

for idx, val := range data {
    fmt.Println(idx, val)
}
Analogi: kayak motong playlist jadi segmen favorit terus diputar ulang sesuai urutan.

D.3 Maps for Key Value Data

Map dipakai untuk data berbasis pasangan key-value. Untuk akses data berdasarkan key, map jauh lebih efisien dibanding cari manual di slice.

skor := map[string]int{
    "Ayu": 92,
    "Bima": 85,
}

skor["Citra"] = 88

if val, ok := skor["Ayu"]; ok {
    fmt.Println("Skor Ayu", val)
}
Analogi: map itu buku kontak, lo cari langsung berdasarkan nama, bukan scroll semua daftar dulu.

D.4 Structs and Methods Basics

Struct dipakai buat model data domain. Method pada struct bikin behavior nempel ke datanya, jadi kode lebih rapi.

type User struct {
    Name string
    Age  int
}

func (u User) Intro() string {
    return fmt.Sprintf("Halo, saya %s", u.Name)
}

u := User{Name: "Raka", Age: 24}
fmt.Println(u.Intro())
Analogi: struct itu kartu profil pemain, method itu aksi yang bisa dilakukan pemain tersebut.

BAB E. Pointers Methods Interfaces

E.1 Pointer Fundamentals

Pointer menyimpan alamat memori. Di Go pointer dipakai supaya perubahan data bisa langsung kena objek asli tanpa copy besar-besaran.

x := 10
p := &x

fmt.Println(x)
fmt.Println(*p)

*p = 20
fmt.Println(x)
Analogi: pointer itu kayak share lokasi, bukan kirim ulang seluruh isi rumah.

E.2 Pointer to Struct and Mutating Data

Untuk update field struct secara efisien, receiver pointer sering dipakai. Ini penting saat objek mulai besar atau sering diubah.

type Account struct {
    Name  string
    Point int
}

func (a *Account) AddPoint(v int) {
    a.Point += v
}

acc := Account{Name: "Nina", Point: 10}
acc.AddPoint(5)
fmt.Println(acc.Point)
Analogi: daripada fotokopi dokumen tiap revisi, mending edit langsung dokumen aslinya.

E.3 Interfaces and Implicit Implementation

Go tidak butuh keyword khusus untuk implement interface. Selama struct punya method yang diminta interface, dia otomatis kompatibel.

type Speaker interface {
    Speak() string
}

type Cat struct{}

func (Cat) Speak() string {
    return "meong"
}

func say(s Speaker) {
    fmt.Println(s.Speak())
}

say(Cat{})
Analogi: selama lo bisa memenuhi syarat masuk, lo boleh ikut pertandingan tanpa daftar label tambahan.

E.4 Polymorphism in Go Design

Polymorphism di Go muncul lewat interface. Ini bikin desain lebih fleksibel dan gampang diuji, karena fungsi cukup bergantung pada perilaku, bukan tipe konkret.

type Payment interface {
    Pay(amount int) string
}

type Cash struct{}

type EWallet struct{}

func (Cash) Pay(amount int) string {
    return fmt.Sprintf("Cash %d", amount)
}

func (EWallet) Pay(amount int) string {
    return fmt.Sprintf("EWallet %d", amount)
}
Analogi: kasir cuma peduli transaksi berhasil, mau bayar tunai atau dompet digital jalurnya tetap bisa diproses.

BAB F. Error Handling Packages and Files

F.1 Error Handling with if err != nil

Go tidak pakai exception sebagai alur utama. Pola umum adalah return error lalu langsung cek dengan if err != nil.

func divide(a int, b int) (int, error) {
    if b == 0 {
        return 0, fmt.Errorf("pembagi tidak boleh nol")
    }
    return a / b, nil
}

res, err := divide(10, 0)
if err != nil {
    fmt.Println("error:", err)
    return
}
fmt.Println(res)
Analogi: tiap langkah kerja ada checklist. Kalau ada masalah, berhenti dulu dan lapor jelas, jangan pura-pura aman.

F.2 Custom Errors and Wrapping Context

Custom error berguna saat domain butuh pesan yang lebih kaya. Error wrapping membantu jejak masalah tetap kebaca sampai level atas.

var ErrNotFound = errors.New("data tidak ditemukan")

func findUser(id int) error {
    return fmt.Errorf("find user %d: %w", id, ErrNotFound)
}
Analogi: bukan cuma bilang "barang hilang", tapi kasih konteks barang apa dan kejadian di mana.

F.3 Packages Modules and Standard Library

Package dipakai buat membagi kode berdasarkan tanggung jawab. Module lewat go.mod ngunci dependency supaya build konsisten lintas mesin.

go mod init contoh-app

go get github.com/google/uuid

go mod tidy
Analogi: package itu pembagian ruangan kerja, module itu buku inventaris resmi semua alat yang dipakai.

F.4 File Handling Basics

Go punya package os dan io yang kuat buat baca tulis file. Untuk operasi simple, os.ReadFile dan os.WriteFile sudah cukup.

content := []byte("belajar golang step by step")
err := os.WriteFile("catatan.txt", content, 0644)
if err != nil {
    fmt.Println(err)
    return
}

data, err := os.ReadFile("catatan.txt")
if err != nil {
    fmt.Println(err)
    return
}
fmt.Println(string(data))
Analogi: tulis file itu kayak simpan nota ke arsip, baca file itu ngambil lagi arsip pas dibutuhkan.

BAB G. Concurrency with Goroutines and Channels

G.1 Goroutines Fundamentals

Goroutine adalah fungsi yang jalan concurrent dengan biaya ringan. Ini alasan utama Go kuat di service yang butuh proses paralel banyak.

func kerja(nama string) {
    fmt.Println("mulai", nama)
    time.Sleep(300 * time.Millisecond)
    fmt.Println("selesai", nama)
}

go kerja("A")
go kerja("B")
time.Sleep(1 * time.Second)
Analogi: satu dapur dengan banyak koki yang bisa masak menu berbeda di waktu bareng.

G.2 Channels for Safe Communication

Channel dipakai buat komunikasi antar goroutine tanpa rebutan data sembarangan. Ini bikin pola concurrent lebih aman dan mudah dipahami.

ch := make(chan string)

go func() {
    ch <- "pesan dari goroutine"
}()

msg := <-ch
fmt.Println(msg)
Analogi: channel itu jalur kirim paket resmi, jadi barang sampai ke tujuan tanpa lempar-lempar liar.

G.3 Buffered Channels and Select

Buffered channel bisa menampung data sebelum dibaca. Select dipakai saat lo menunggu banyak channel sekaligus.

ch1 := make(chan string, 1)
ch2 := make(chan string, 1)

ch1 <- "dari ch1"
ch2 <- "dari ch2"

select {
case m1 := <-ch1:
    fmt.Println(m1)
case m2 := <-ch2:
    fmt.Println(m2)
default:
    fmt.Println("belum ada pesan")
}
Analogi: select itu seperti operator call center yang siap terima telepon dari beberapa jalur masuk.

G.4 Common Concurrency Pitfalls

Yang sering bikin bug: deadlock, goroutine leak, dan race condition. Biasakan desain alur channel jelas dan tutup channel saat memang selesai dipakai.

go test -race ./...
  • Hindari kirim ke channel yang tidak pernah dibaca.
  • Gunakan context untuk cancel operasi panjang.
  • Audit ownership data yang diakses banyak goroutine.
Analogi: kalau semua orang ngobrol bareng tanpa aturan giliran, rapat pasti chaos.

BAB H. HTTP JSON and Testing

H.1 Building Simple HTTP Server

Package net/http cukup buat bikin server sederhana tanpa framework tambahan. Cocok buat paham fondasi web di Go.

http.HandleFunc("/ping", func(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintln(w, "pong")
})

http.ListenAndServe(":8080", nil)
Analogi: net/http itu dapur dasar, lo sudah bisa buka warung tanpa dekor mewah dulu.

H.2 Routing and JSON Response

Endpoint API umum akan kirim response JSON. Encoding decoding JSON jadi skill wajib backend Go.

type Resp struct {
    Message string `json:"message"`
}

http.HandleFunc("/hello", func(w http.ResponseWriter, r *http.Request) {
    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(Resp{Message: "halo"})
})
Analogi: JSON itu format paket standar, jadi sistem beda-beda tetap bisa saling paham isi kiriman.

H.3 JSON Decode with Struct Tags

Struct tag memastikan field Go bisa match dengan key JSON yang biasanya snake_case atau lower case.

type User struct {
    FullName string `json:"full_name"`
    Age      int    `json:"age"`
}

raw := []byte(`{"full_name":"Rani","age":24}`)
var u User
err := json.Unmarshal(raw, &u)
if err != nil {
    fmt.Println(err)
    return
}
fmt.Println(u.FullName, u.Age)
Analogi: struct tag itu label penerjemah biar format luar dan format internal bisa nyambung rapi.

H.4 Basic Unit Testing in Go

Testing di Go ringan dan langsung kepake lewat package testing. Biasakan nulis test kecil sejak awal biar refactor lebih aman.

// file: math_test.go
func TestTambah(t *testing.T) {
    got := tambah(2, 3)
    want := 5

    if got != want {
        t.Fatalf("got %d want %d", got, want)
    }
}

// jalankan:
// go test ./...
Analogi: test itu quality check sebelum barang dikirim ke pelanggan.

BAB I. Suggested Path and Practice

I.1 Suggested Learning Path

Biar belajarnya gak loncat-loncat, ini urutan yang paling masuk akal:

  1. Basics dulu: intro, syntax, variable, control flow, loop.
  2. Naik ke function, slice, map, struct, interface.
  3. Kuatkan error handling dan package module.
  4. Masuk concurrency pelan-pelan sampai paham pola.
  5. Tutup dengan HTTP, JSON, dan testing.
Analogi: jangan langsung main liga utama, bangun stamina dari latihan dasar yang konsisten.

I.2 Practice Project Ideas

Project kecil yang realistis dan efektif buat ngunci konsep:

  • CLI calculator dengan validasi input.
  • File processor untuk baca dan olah text log.
  • REST API CRUD sederhana.
  • Concurrent worker system untuk proses job batch.
# contoh jalur latihan
week1: basic + function
week2: slice map struct
week3: interface + error handling
week4: goroutine channel + mini API
Analogi: skill ngoding itu kebentuk dari repetisi drill, bukan dari nonton teori doang.

I.3 Practical Tips for Fast Improvement

  • Fokus dulu kuasai slice, interface, dan error handling.
  • Tulis kode sedikit tapi rutin tiap hari.
  • Pahami dulu standar library sebelum buru-buru framework.
  • Biasakan profiling dan test saat project mulai gede.

Kalau konsisten di fondasi ini, transisi ke production-grade service bakal jauh lebih mulus.

Analogi: pemain yang latihan dasar bener biasanya lebih stabil pas tanding dibanding yang cuma jago trik.
Kesimpulan: Kekuatan Go ada di kombinasi simplicity, performa, dan concurrency. Kuasai fondasi dulu, lalu naik bertahap ke API dan project concurrent. Dengan pola latihan terarah, skill Go lo bakal cepat matang dan siap dipakai di kerjaan nyata.
