BAB A. JavaScript Basics

A.1 JavaScript Overview

JavaScript adalah bahasa pemrograman utama buat interaksi di web. HTML itu rangka, CSS itu style, JavaScript itu mesin yang bikin semuanya bergerak. Tanpa JS, web lo bisa tampil tapi gak interaktif.

Hari ini JavaScript gak cuma hidup di browser. Lo juga bisa pakai JS di backend (Node.js), desktop app, mobile app, automation, sampai tooling. Jadi belajar JavaScript itu ibarat belajar bahasa umum yang bisa dipakai di banyak arena.

  • Client-side: berjalan di browser user.
  • Server-side: bisa jalan di server pakai Node.js.
  • Kelebihan: ekosistem luas, fleksibel, cepat dipelajari.
  • Keterbatasan: kalau code berantakan, gampang jadi susah maintain.
  • Tools: browser devtools, Node.js, VS Code.
Analogi tongkrongan: JavaScript itu kayak kru operasional di tempat nongkrong. Tempatnya udah ada, dekor udah cakep, tapi yang bikin pesanan jalan dan customer dilayanin ya kru ini.

A.2 Syntax Dasar

Di level syntax, lo belajar aturan nulis: case-sensitive, komentar, semicolon, whitespace, dan gaya penulisan yang konsisten. Ini keliatan sepele, tapi jadi penentu apakah code enak dibaca tim atau bikin pusing.

  • Case-sensitive: namaUser beda sama NamaUser.
  • Komentar: tulis niat kode lo biar tim ngerti konteks.
  • Semicolon: technically optional, tapi konsisten itu wajib.
  • Formatting: indentasi rapi bikin debugging lebih cepat.
// komentar satu baris
const nama = "Bimo";
let umur = 21;

if (umur >= 18) {
  console.log(nama + " sudah cukup umur");
}

Pelajaran penting di sini: kode yang "jalan" belum tentu kode yang "enak dirawat". Biasain dari awal nulis kode yang rapi, bukan nunggu project rusak dulu.

A.3 Enabling JavaScript di Browser

Dulu tiap browser punya perilaku agak beda, makanya bagian enabling penting buat paham konteks historis. Sekarang browser modern udah jauh lebih kompatibel, tapi tetap perlu aware kalau user matiin JS, beberapa fitur gak jalan.

Artinya, UI lo sebisa mungkin tetap kasih fallback minimal. Misalnya tombol submit form tetap bisa jalan secara default walau script error.

Mindset penting: JavaScript nambah pengalaman user, bukan jadi satu-satunya fondasi supaya halaman bisa dipakai.

A.4 Placement Script

Script bisa ditaruh di head, body, atau file eksternal. Praktik terbaik sekarang: file eksternal plus defer atau di akhir body.

  • Load lebih optimal.
  • Codebase lebih rapih.
  • Lebih gampang cache dan maintain.
  • Minim risiko element belum kebaca waktu script dijalankan.
<script src="app.js" defer></script>

Pakai file eksternal juga bikin project gampang dipisah: HTML fokus struktur, CSS fokus tampilan, JS fokus logic.

A.5 Variables dan Datatypes

Variabel adalah penampung data. Di JS modern, gunakan const untuk nilai tetap dan let untuk nilai yang berubah.

  • Tipe data dasar: string, number, boolean, null, undefined.
  • Tipe data kompleks: object, array, function.
  • Scope penting: global, function, block scope.
  • Hindari var untuk kode baru karena perilakunya lebih rawan bug (hoisting + function scope).
const toko = "Ngopi Santuy";
let stokKopi = 10;

if (stokKopi > 0) {
  const status = "Ready"; // block scope
  console.log(status);
}

stokKopi = stokKopi - 1;

Kalau lo paham scope dari awal, lo bakal jauh lebih minim bug aneh kayak nilai ketimpa dari tempat yang gak ketebak.

A.6 Operators

Operator dipakai buat hitung, bandingin, dan ambil keputusan logika. Yang sering kepakai: arithmetic, comparison, logical, assignment.

const harga = 20000;
const qty = 3;
const total = harga * qty;
const diskon = total > 50000;

console.log(total, diskon);
  • Arithmetic: +, -, *, /, %.
  • Comparison: ===, !==, >, <.
  • Logical: &&, ||, !.
  • Assignment: =, +=, -=.

Pakai === sebagai default biar perbandingan lebih ketat dan prediktif.

A.7 If-Else dan Switch

Ini fondasi pengambilan keputusan. If-else cocok untuk kondisi fleksibel, switch cocok untuk banyak pilihan nilai diskrit.

const role = "admin";

if (role === "admin") {
  console.log("Akses penuh");
} else if (role === "staff") {
  console.log("Akses terbatas");
} else {
  console.log("Akses tamu");
}
Analogi tongkrongan: "kalau hujan pindah indoor" itu if. "kalau menu kopi pilih A, kalau teh pilih B" itu switch.

Tips praktis: kalau kondisi makin panjang dan ribet, pecah ke function biar kebaca dan gampang dites.

A.8 Loops dan Loop Control

Loop bikin lo gak perlu nulis kode berulang. Gunakan while, do...while, for, for...in sesuai kebutuhan. Gunakan break/continue dengan hati-hati supaya alur tetap jelas.

  • for: cocok kalau jumlah iterasi udah jelas.
  • while: cocok kalau berhenti berdasarkan kondisi dinamis.
  • for...of: enak buat iterasi array modern.
  • break/continue: pakai seperlunya, jangan bikin alur susah dibaca.
const antrian = ["A", "B", "C", "D"];

for (const kode of antrian) {
  if (kode === "C") continue;
  console.log("Layani", kode);
}

A.9 Functions

Function itu paket kerja reusable. Lo bisa bikin parameter, return value, nested function, dan pola deklarasi yang beda sesuai kebutuhan.

function hitungPajak(nilai, persen) {
  return nilai * (persen / 100);
}

const pajak = hitungPajak(100000, 11);
console.log(pajak);

Prinsip yang sering dipakai di project nyata:

  • Satu function satu tanggung jawab.
  • Nama function jelas sesuai aksi: hitungTotal(), validasiEmail().
  • Hindari function terlalu panjang, pecah kalau udah campur banyak urusan.

A.10 Events

Events bikin halaman merespons aksi user: click, submit, mouseover, dan event HTML5 lain. Ini kunci UX modern.

const form = document.querySelector("#form-order");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  console.log("Order diproses");
});

Poin pentingnya: event-driven programming bikin app terasa hidup. Tapi jangan lupa manage event listener biar gak bocor memory di komponen yang sering mount/unmount.

A.11 Cookies, Redirect, Dialog, Print

Topik ini ngajarin interaksi browser level klasik: simpan state ringan pakai cookie, pindah halaman pakai redirect, interaksi cepat pakai alert/confirm/prompt, dan print halaman saat dibutuhkan.

  • Cookie: simpan data kecil (misal preferensi user), bukan data sensitif mentah.
  • Redirect: pakai saat alur butuh pindah halaman setelah aksi tertentu.
  • Dialog: cocok buat konfirmasi cepat, tapi jangan berlebihan karena ganggu UX.
  • Print: useful untuk invoice/receipt/report sederhana.
Analogi tongkrongan: cookie itu catatan kecil pelanggan favorit, redirect itu ngarahin customer ke kasir yang benar, dialog itu nanya "yakin pesan ini?" sebelum diproses.

A.12 Ringkasan Basic

Kalau bagian basic ini lo kuasai, lo udah punya pondasi kokoh. Jangan buru-buru lompat ke framework kalau syntax, flow, function, event, dan manipulasi data sederhana belum solid.

Checklist siap lanjut ke BAB B:

  1. Lo nyaman baca/tulis kode JS tanpa lihat contekan terus.
  2. Lo paham kapan pakai if, switch, loop, dan function.
  3. Lo bisa handle event form dan validasi dasar.
  4. Lo ngerti bedanya bug syntax, logic, dan runtime di level dasar.

BAB B. JavaScript Objects dan Tipe Data Built-in

B.1 Konsep Object di JavaScript

Object adalah format data paling penting di JavaScript. Kalau di dunia nyata lo punya kartu profil orang (nama, role, status), di JS itu direpresentasikan jadi object dengan pasangan key: value.

Di dalam object, lo bisa simpan:

  • Property: data, misalnya nama, umur.
  • Method: fungsi milik object, misalnya sapa().
const user = {
  nama: "Rina",
  role: "admin",
  aktif: true,
  sapa() {
    return `Halo, saya ${this.nama}`;
  }
};

console.log(user.nama);      // Rina
console.log(user.sapa());    // Halo, saya Rina
Analogi tongkrongan: object itu kayak profil member komunitas. Satu orang, tapi datanya banyak dan aksinya juga ada.

B.2 Number: Angka dan Formatting

Number dipakai buat seluruh operasi angka: total belanja, diskon, rating, dan statistik. Yang penting bukan cuma ngitung, tapi juga format output supaya enak dibaca user.

const harga = 12345.6789;

console.log(harga.toFixed(2));                  // 12345.68
console.log(harga.toLocaleString("id-ID"));    // 12.345,679

const qty = Number("3");
const total = harga * qty;
console.log(total);

console.log(Number.isNaN(Number("abc")));      // true

Tips: untuk tampilan uang, biasanya pakai toLocaleString. Untuk perhitungan backend, simpan nilai mentahnya.

B.3 Boolean: Logika True/False

Boolean itu pondasi keputusan di program. Hampir semua validasi dan permission ujungnya balik ke true atau false.

const inputEmail = "user@mail.com";
const isValidEmail = inputEmail.includes("@");

const role = "admin";
const isAdmin = role === "admin";
const isActive = true;

if (isValidEmail && isAdmin && isActive) {
  console.log("Akses dashboard admin");
} else {
  console.log("Akses ditolak");
}

Hindari logika boolean yang terlalu panjang dalam satu baris. Pecah ke variabel kecil biar gampang dibaca dan dites.

B.4 String: Manipulasi Teks

String methods dipakai terus di project nyata: validasi input, format nama user, parsing query, sampai bersihin data sebelum disimpan.

const kalimat = "belajar javascript itu seru";

console.log(kalimat.toUpperCase());
console.log(kalimat.indexOf("javascript"));
console.log(kalimat.replace("seru", "menantang"));
console.log(kalimat.split(" "));

const email = "  USER@MAIL.COM  ";
console.log(email.trim().toLowerCase()); // user@mail.com

Langkah klasik saat terima input teks: trim() dulu, baru validasi atau simpan.

B.5 Array dan Array of Objects

Array itu object khusus untuk data berurutan. Sementara array of objects berarti isi array-nya adalah object. Ini pola paling umum di aplikasi real karena data biasanya berbentuk list record.

const mahasiswa = [
  { nama: "Budi", nilai: 70 },
  { nama: "Siti", nilai: 85 },
  { nama: "Andi", nilai: 90 },
  { nama: "Rina", nilai: 60 }
];

const lulus = mahasiswa.filter((m) => m.nilai >= 75);
const namaLulus = lulus.map((m) => m.nama);
const rata2 = mahasiswa.reduce((acc, m) => acc + m.nilai, 0) / mahasiswa.length;

console.log(namaLulus); // ["Siti", "Andi"]
console.log(rata2);
Analogi tongkrongan: array itu kayak antrean orang. Array of objects itu antrean orang lengkap sama data profilnya.

B.6 Date: Waktu dan Tanggal

Date dipakai untuk timestamp, deadline, laporan harian, dan histori transaksi. Bagian yang sering bikin salah adalah format tanggal dan timezone, jadi cek output dengan teliti.

const sekarang = new Date();
const deadline = new Date("2026-04-01T17:00:00");

console.log(sekarang.toLocaleString("id-ID"));
console.log(deadline.getDate());
console.log(deadline.getMonth() + 1);
console.log(deadline.getFullYear());

const selisihMs = deadline - sekarang;
const sisaHari = Math.ceil(selisihMs / (1000 * 60 * 60 * 24));
console.log("Sisa hari ke deadline:", sisaHari);

B.7 Math: Operasi Numerik

Math object dipakai untuk operasi angka yang gak disediakan operator biasa: pembulatan, random, pangkat, akar, dan fungsi trigonometri.

const nilai = 19.7;

console.log(Math.floor(nilai));
console.log(Math.ceil(nilai));
console.log(Math.round(nilai));
console.log(Math.pow(2, 5));
console.log(Math.sqrt(144));

const angkaAcak1to10 = Math.floor(Math.random() * 10) + 1;
console.log(angkaAcak1to10);

B.8 Regular Expression (RegExp)

Regular expression dipakai buat validasi pola teks: email, nomor HP, kode transaksi, dan pencarian string tertentu. Powerful, tapi tetap bikin pattern yang readable.

const emailPattern = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
const kodeOrderPattern = /^ORD-\d{4}$/;

console.log(emailPattern.test("halo@mail.com")); // true
console.log(emailPattern.test("halo.mail.com")); // false

console.log(kodeOrderPattern.test("ORD-2026")); // true
console.log(kodeOrderPattern.test("ORDER-26")); // false
Analogi tongkrongan: RegExp itu kayak satpam pintu yang ngecek pola dress code. Yang sesuai aturan boleh masuk, yang gak sesuai ditolak.

B.9 DOM: Manipulasi Elemen HTML

DOM adalah jembatan antara JavaScript dan HTML. Lo bisa pilih elemen, ubah konten, kasih class, dan handle event user secara real-time.

const title = document.querySelector("h1");
const btnTheme = document.querySelector("#btn-theme");

title.textContent = "Belajar DOM Itu Wajib";

btnTheme.addEventListener("click", () => {
  document.body.classList.toggle("active");
  title.classList.toggle("highlight");
});

Pola ini jadi pondasi semua framework UI modern: state berubah, UI ikut berubah.

B.10 Ringkasan BAB B

BAB B ngajarin lo cara JavaScript ngelola data dari bentuk sederhana sampai kompleks. Kalau bagian ini kuat, lo bakal lebih pede masuk React/Vue karena konsep dasarnya sudah kebangun.

const peserta = [
  { nama: "Nina", nilai: 88, aktif: true },
  { nama: "Rafi", nilai: 72, aktif: true }
];

const tanggalBelajar = new Date().toLocaleDateString("id-ID");
const siapLanjut = peserta
  .filter((p) => p.aktif)
  .map((p) => `${p.nama}: ${Math.round(p.nilai)}`);

console.log("Tanggal:", tanggalBelajar);
console.log("Peserta aktif:", siapLanjut.join(", "));

BAB C. JavaScript Advanced

C.1 Errors dan Exceptions

Di production, error itu pasti ada. Bedain jenis error dan tangani dengan strategi yang tepat.

  • Syntax error: kesalahan penulisan kode.
  • Runtime error: error saat aplikasi berjalan.
  • Logical error: aplikasi jalan, tapi hasilnya salah.
try {
  const data = JSON.parse("invalid");
  console.log(data);
} catch (err) {
  console.error("Format JSON salah:", err.message);
} finally {
  console.log("Proses selesai");
}

C.2 Form Validation

Validasi form harus berlapis: client-side buat UX cepat, server-side buat keamanan. Validasi mencakup required field, format data, panjang input, dan sanitasi.

const form = document.querySelector("#registerForm");
const emailInput = document.querySelector("#email");

form.addEventListener("submit", (e) => {
  const email = emailInput.value.trim().toLowerCase();
  const emailPattern = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;

  if (!emailPattern.test(email)) {
    e.preventDefault();
    alert("Format email belum valid");
    return;
  }

  console.log("Validasi client-side lolos");
});

C.3 Animation

Animation bikin UI lebih hidup, tapi harus tetap fungsional. Fokus ke transisi yang membantu user paham perubahan state, bukan sekadar hiasan.

const box = document.querySelector(".card");
const btn = document.querySelector("#btnAnimate");

btn.addEventListener("click", () => {
  box.classList.add("fade-in");

  setTimeout(() => {
    box.classList.remove("fade-in");
  }, 500);
});

/* CSS contoh
.fade-in {
  animation: fadeIn 0.5s ease;
}
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(8px); }
  to   { opacity: 1; transform: translateY(0); }
}
*/

C.4 Multimedia

JavaScript juga bisa kontrol audio/video: play, pause, seek, volume, dan event media. Ini penting untuk aplikasi edukasi, streaming, dan konten interaktif.

const video = document.querySelector("#videoBelajar");
const btnPlay = document.querySelector("#btnPlay");
const btnPause = document.querySelector("#btnPause");

btnPlay.addEventListener("click", () => video.play());
btnPause.addEventListener("click", () => video.pause());

video.addEventListener("timeupdate", () => {
  console.log("Detik berjalan:", Math.floor(video.currentTime));
});

C.5 Debugging

Debugging adalah skill inti developer. Gunakan console, breakpoint, watch, call stack, dan network tab untuk ngelacak akar masalah.

  1. Reproduce issue dengan langkah yang jelas.
  2. Isolasi area masalah sekecil mungkin.
  3. Perbaiki akar masalah, bukan tempelan sementara.
  4. Retest seluruh alur terkait.
function hitungDiskon(total, persen) {
  console.log("Input:", { total, persen });

  if (typeof total !== "number" || typeof persen !== "number") {
    throw new Error("total dan persen harus number");
  }

  const hasil = total - (total * persen / 100);
  console.log("Output:", hasil);
  return hasil;
}

console.log(hitungDiskon(100000, 10));
Analogi tongkrongan: debugging itu kayak benerin speaker yang suaranya pecah. Lo cek kabel, power, mixer, sampe sumber noise-nya ketemu. Bukan asal ketok-ketok alat.

C.6 Ringkasan Advanced

Di level advanced, mindset lo berubah dari "bisa jalan" jadi "bisa diandalkan". Lo mulai mikirin stability, maintainability, dan quality saat aplikasi dipakai banyak user.

const materiAdvanced = ["error handling", "validation", "animation", "multimedia", "debugging"];

const checklist = materiAdvanced.map((item, i) => `${i + 1}. ${item}`);
console.log("Checklist BAB C:");
console.log(checklist.join("\n"));
Kesimpulan Full Materi: jalur belajar paling aman adalah kuatin Basic dulu, lanjut Objects sampai terbiasa ngolah data + DOM, baru masuk Advanced buat bikin aplikasi yang tahan banting. Urutan ini bikin progres lo lebih stabil dan minim frustrasi.
